## 1、String

在 C++ 中，`std::string` 是一个非常常用的类，定义在 `<string>` 头文件中，用于处理字符串。它提供了丰富的成员函数和操作接口，让你可以方便地进行字符串的创建、修改、查找、拼接等操作。

下面是一些常用 `std::string` 接口的分类整理（以 C++11 及以后标准为准）：

---

### 🔹1.1 构造与赋值

|接口|说明|
|---|---|
|`string()`|默认构造：空字符串|
|`string(const char* s)`|用 C 风格字符串初始化|
|`string(const string& s)`|拷贝构造|
|`string(size_t n, char c)`|构造 n 个字符 c|
|`string(const char* s, size_t n)`|前 n 个字符构造|
|`string(initializer_list)`|列表初始化（如 `"abc"`）|
|`str = "hello"`|赋值运算符|

```cpp
std::string s1;
std::string s2("Hello");
std::string s3(5, 'a'); // "aaaaa"
std::string s4(s2);     // 拷贝
```

---

### 🔹1.2 访问元素

|接口|说明|
|---|---|
|`s[i]`|返回第 i 个字符（不检查越界）|
|`s.at(i)`|返回第 i 个字符（越界抛出 `out_of_range` 异常）|
|`front()`|第一个字符引用|
|`back()`|最后一个字符引用|
|`c_str()`|返回 C 风格字符串（const char*）|
|`data()`|返回底层数据指针（C++17 起可用）|

```cpp
std::string s = "hello";
char ch = s[1];        // 'e'
char ch2 = s.at(1);    // 同上，安全
s.front() = 'H';       // 改第一个字符
const char* cstr = s.c_str();
```

---

### 🔹1.3 长度与容量

|接口|说明|
|---|---|
|`size()` / `length()`|字符数（返回 `size_t`）|
|`empty()`|是否为空（true/false）|
|`capacity()`|当前分配的内存大小（可容纳多少字符）|
|`reserve(n)`|预留至少 n 个字符空间|
|`shrink_to_fit()`|请求缩减容量至实际长度（非强制）|

```cpp
std::string s = "hello";
std::cout << s.size();      // 5
std::cout << s.empty();     // false
s.reserve(100);             // 预留100字节
s.shrink_to_fit();          // 尝试缩小
```

---

### 🔹1.4 增删改查操作

✅ 拼接与添加

|接口|说明|
|---|---|
|`+=`|追加字符串/字符|
|`append()`|添加子串或字符|
|`insert(pos, str)`|在 pos 插入字符串|
|`push_back(c)`|添加单个字符|

```cpp
s += " world";
s.append("!!!");           // 拼接
s.insert(5, "beautiful "); // 插入到位置5
s.push_back('!');          // 添加!
```

❌ 删除

|接口|说明|
|---|---|
|`erase(pos, len)`|删除从 pos 开始的 len 个字符|
|`pop_back()`|删除最后一个字符|
|`clear()`|清空字符串|

```cpp
s.erase(5, 7);   // 删除从索引5开始的7个字符
s.pop_back();    // 删除最后一位
s.clear();       // 变为空
```

🔁 替换

|接口|说明|
|---|---|
|`replace(pos, len, str)`|替换指定范围为新字符串|
|`swap(str)`|交换两个字符串内容|

```cpp
s.replace(0, 5, "Hi"); // "Hi world" → "Hi world"
std::string other = "test";
s.swap(other);
```

---

### 🔹1.5 查找与比较

|接口|说明|
|---|---|
|`find(str, pos)`|查找子串首次出现的位置（找不到返回 `std::string::npos`）|
|`rfind(str, pos)`|从后往前找|
|`find_first_of(str, pos)`|找第一个在 str 中出现的字符|
|`find_last_of(str, pos)`|找最后一个在 str 中出现的字符|
|`find_first_not_of(str, pos)`|找第一个不在 str 中的字符|
|`find_last_not_of(str, pos)`|找最后一个不在 str 中的字符|
|`compare(str)`|字典序比较（<0, ==0, >0）|

```cpp
std::string s = "hello world";
size_t pos = s.find("world"); // 返回6
if (pos != std::string::npos) {
    std::cout << "found at " << pos << "\n";
}
```

---

### 🔹1.6 子串提取

|接口|说明|
|---|---|
|`substr(pos, len)`|提取从 pos 开始、长度为 len 的子串|

```cpp
std::string sub = s.substr(6, 5); // "world"
```

---

### 🔹1.7 其他实用接口

|接口|说明|
|---|---|
|`operator==`, `!=`, `<`, `<=`, etc.|支持比较操作|
|`getline(cin, s)`|从输入流读取一行（需包含 ``）|
|`stoi(s)`, `stol`, `stod`, `stof`|字符串转数字（C++11）|
|`to_string(val)`|数字转字符串（C++11）|

```cpp
int num = std::stoi("123");
std::string str = std::to_string(456);
std::getline(std::cin, s); // 输入一行
```

---

📌 小贴士

- `std::string::npos` 是一个特殊值，表示“未找到”。
- 使用 `at()` 更安全，适合调试阶段。
- `c_str()` 返回的是临时指针，不要保存太久，尤其在修改字符串后可能失效。
- `data()` 在 C++17 后是安全的，可用于传给 C 函数。

---

🔍 补充说明：

- `find_first_not_of(' ')`：找第一个**不是空格**的字符。
- `find_last_not_of(' ')`：找最后一个**不是空格**的字符。
- 如果整个字符串都是空格，比如 `" "`，那么 `find_first_not_of(' ')` 会返回 `std::string::npos`（表示没找到），这时你需要判断是否为空。

---

✅ 安全写法（带边界判断）：

```cpp
std::string trim(const std::string& s) {
    size_t start = s.find_first_not_of(' ');
    if (start == std::string::npos) return "";  // 全是空格

    size_t end = s.find_last_not_of(' ');
    return s.substr(start, end - start + 1);
}
```

## 2、queue

`std::queue` 基本特性

- ​**功能**​：先进先出（FIFO）数据结构。
- ​**底层实现**​：通常基于 `deque` 或 `list`，支持高效插入和删除操作。
- ​**线程安全**​：默认**不是线程安全**的，需自行加锁保护。
- ​**容量限制**​：无显式容量限制（取决于底层容器）。

---

📚 接口列表（成员函数）

### 2.1 构造函数

|函数|说明|
|---|---|
|`queue()`|构造一个空队列|
|`queue(const queue& other)`|复制构造（拷贝另一个队列）|
|`queue(queue&& other)`|移动构造（移动另一个队列）|

> ⚠️ 注意：不能直接构造 `queue<T>` 时传入 `initializer_list`，除非使用 `std::queue` 的构造函数（如 `queue(container)`）。

---

### 2.2 赋值与移动

|函数|说明|
|---|---|
|`queue& operator=(const queue& other)`|拷贝赋值|
|`queue& operator=(queue&& other)`|移动赋值|

---

### 2.3 元素访问（只读）

|函数|说明|
|---|---|
|`front()`|返回队头元素（​**不移除**​）|
|`back()`|返回队尾元素（​**不移除**​）|

> ❗ 抛出异常：如果队列为空，则调用 `front()` 或 `back()` 会抛出 `std::queue_empty` 异常（实际是 `std::runtime_error` 或未定义行为，​**建议先判空**​）

---

### 2.4 操作（修改队列）

|函数|说明|
|---|---|
|`push(const T& value)`|将元素添加到队尾|
|`push(T&& value)`|移动构造添加到队尾|
|`pop()`|移除队头元素（​**不返回值**​）|

> ⚠️ `pop()` 在空队列上调用会导致未定义行为（UB）！

---

### 2.5 查询（状态检查）

|函数|说明|
|---|---|
|`empty()`|返回 `true` 表示队列为空|
|`size()`|返回队列中元素个数|

---

### 2.6 交换（swap）

|函数|说明|
|---|---|
|`void swap(queue& other)`|与另一个队列交换内容（常用于优化性能）|

> 该操作是​**O(1)**​，且​**不抛异常**​。

---

📌 使用示例

```cpp
#include <iostream>
#include <queue>

int main() {
    std::queue<int> q;

    // 添加元素
    q.push(10);
    q.push(20);
    q.push(30);

    // 查看队头和队尾
    std::cout << "Front: " << q.front() << std::endl;  // 输出 10
    std::cout << "Back: " << q.back() << std::endl;   // 输出 30

    // 移除队头
    q.pop();

    std::cout << "After pop, front: " << q.front() << std::endl; // 输出 20

    // 判断是否为空
    if (!q.empty()) {
        std::cout << "Queue has " << q.size() << " elements." << std::endl;
    }

    return 0;
}
查看全部
```

---

⚠️ 注意事项

|项目|说明|
|---|---|
|**不支持迭代器**|`std::queue` 没有迭代器，无法像 `vector` 那样遍历|
|**不能随机访问**|只能访问头和尾|
|**无下标操作**|不存在 `q[0]` 这种语法|
|**线程安全问题**|多线程下必须加锁（例如 `std::mutex`）|
|**性能**|`push()` 和 `pop()` 是 O(1)，适合任务调度、BFS 等场景|

---

🔄 常见替代方案（当需要更多功能时）

|场景|推荐容器|
|---|---|
|需要遍历|`std::deque`, `std::vector`|
|需要双端访问|`std::deque`|
|需要线程安全|`std::queue` + `std::mutex`|
|需要优先级|`std::priority_queue`|

---

### 2.7 ✅ 总结一句话：

> `std::queue` 是一个简单高效的 FIFO 容器，提供 `push`, `pop`, `front`, `back`, `empty`, `size` 等基本接口，适用于任务队列、BFS、缓冲区等典型场景。

## 3、Map

🗂️ `std::map` 速查卡（C++ STL）

> 基于红黑树实现，键唯一、自动排序（升序），支持高效查找、插入、删除。

```cpp
#include <map>
std::map<Key, Value, Compare = std::less<Key>, Allocator = std::allocator<std::pair<const Key, Value>>>
```

---

### 🔧3.1 构造函数

|函数|说明|
|---|---|
|`map()`|构造空 map|
|`map(const map& other)`|复制构造|
|`map(map&& other)`|移动构造|
|`map(InputIterator first, InputIterator last)`|用范围初始化|
|`map(InputIterator first, InputIterator last, const Compare& comp)`|自定义比较器|
|`map(const allocator_type& alloc)`|指定分配器|

---

### 🔄 3.2 赋值与交换

|函数|说明|
|---|---|
|`map& operator=(const map& other)`|复制赋值|
|`map& operator=(map&& other)`|移动赋值|
|`void swap(map& other)`|交换两个 map（O(1)）|

---

🕹️ 三、迭代器

|函数|返回类型|说明|
|---|---|---|
|`begin()`|`iterator`|第一个元素|
|`end()`|`iterator`|末尾（不包含）|
|`rbegin()`|`reverse_iterator`|逆序开始|
|`rend()`|`reverse_iterator`|逆序结束|
|`cbegin()`|`const_iterator`|常量迭代器|
|`cend()`|`const_iterator`|常量末尾|
|`crbegin()`|`const_reverse_iterator`|常量逆序开始|
|`crend()`|`const_reverse_iterator`|常量逆序结束|

> ✅ 推荐遍历方式：

```cpp
for (const auto& [k, v] : m) {
    std::cout << k << " -> " << v << "\n";
}
```

---

### 📊 3.3 容量

|函数|返回值|说明|
|---|---|---|
|`size()`|`size_type`|当前元素个数|
|`empty()`|`bool`|是否为空|
|`max_size()`|`size_type`|最大可能容量|
|`reserve(n)`|void|预留空间（非必须，map 不常扩容）|
|`shrink_to_fit()`|void|释放多余内存（C++11+）|

---

### 🔍 3.4 访问元素

|函数|说明|
|---|---|
|`operator[](const Key& k)`|获取 value（若 key 不存在则插入默认值）⚠️危险|
|`at(const Key& k)`|获取 value（越界抛 `std::out_of_range`）✅安全|
|`find(const Key& k)`|返回迭代器（未找到返回 `end()`）✅推荐|
|`count(const Key& k)`|返回 0 或 1（map 中不会超过 1）|
|`equal_range(const Key& k)`|返回 `[lower_bound, upper_bound)` 区间（可用于多值场景）|

> ❗ 注意：
> 
> - `operator[]` 会隐式插入 `Key()` 和 `Value()`，可能引起意外行为。
> - 用 `at()` 替代 `operator[]` 可提高安全性。

---

### ➕ 3.5 插入操作

|函数|返回值|说明|
|---|---|---|
|`insert(value_type val)`|`pair`|成功插入返回 `true`|
|`insert(iterator hint, value_type val)`|`iterator`|建议提供提示位置|
|`insert(InputIterator first, InputIterator last)`|void|插入范围|
|`emplace(Args&&... args)`|`pair`|直接构造元素（更高效）|
|`emplace_hint(iterator hint, Args&&... args)`|`iterator`|带提示的 emplace|

> ✅ 推荐写法：

```cpp
m.insert({key, value});
// 或更高效：
m.emplace(key, value);
```

---

### 🗑️ 3.6 删除操作

|函数|说明|
|---|---|
|`erase(const Key& k)`|删除指定 key（返回删除数量）|
|`erase(iterator pos)`|删除指定位置|
|`erase(iterator first, iterator last)`|删除区间|
|`clear()`|清空所有元素|

> 💡 示例：

```cpp
m.erase(5);                  // 删除 key=5
m.erase(m.find(10));         // 安全删除（先 find）
```

---

### 🔎 3.7 查找辅助函数

|函数|返回值|说明|
|---|---|---|
|`lower_bound(const Key& k)`|`iterator`|第一个 ≥ k 的元素|
|`upper_bound(const Key& k)`|`iterator`|第一个 > k 的元素|
|`equal_range(const Key& k)`|`pair`|`[lower, upper)` 范围|

> ✅ 用途：查找范围、统计频次、分组处理

```cpp
auto range = m.equal_range(5);
for (auto it = range.first; it != range.second; ++it) {
    std::cout << it->second << "\n";
}
```

---

### 🛠️ 3.8 比较与分配器

|函数|返回值|说明|
|---|---|---|
|`key_comp()`|`Compare`|获取键比较函数|
|`value_comp()`|`Compare`|获取 value 比较函数（通常无意义）|
|`get_allocator()`|`Allocator`|获取分配器对象|

> 💡 自定义排序示例：

```cpp
struct DescCompare {
    bool operator()(int a, int b) const {
        return a > b; // 降序
    }
};
std::map<int, std::string, DescCompare> m;
```

---

### ✅ 3.9 最佳实践建议

|场景|推荐做法|
|---|---|
|查找是否存在|`if (m.find(k) != m.end())`|
|安全获取值|`m.at(k)`（异常处理）|
|插入新元素|`m.insert({k, v})` 或 `m.emplace(k, v)`|
|遍历所有元素|`for (const auto& [k, v] : m)`|
|避免隐式插入|不要用 `m[k]` 除非确定 key 存在|
|性能优化|使用 `emplace()` 替代 `insert()`|

---

📌 附：`map` vs `unordered_map` 快速对比

|特性|`std::map`|`std::unordered_map`|
|---|---|---|
|底层结构|红黑树|哈希表|
|时间复杂度|O(log n)|平均 O(1)，最坏 O(n)|
|是否有序|✅ 是（按键排序）|❌ 否|
|内存开销|较高|较低|
|适用场景|需要顺序遍历、范围查询|快速查找、高频插入/删除|

---

✨ **小贴士：**

> “别让 `operator[]` 悄悄帮你插入数据！”  
> —— 用 `find()` + `at()` 更安全，代码更健壮！

## 4、std::pair

> C++ 标准库中的键值对容器，常用于返回多个值、映射关系等场景。

---

### 4.1 🔧 基本定义

```cpp
#include <utility>  // 必需头文件
std::pair<类型1, 类型2> 变量名;
```

✅ 示例：

```cpp
std::pair<std::string, int> student{"Alice", 20};
std::pair<double, bool> result{3.14, true};
```

---

### 4.2 🎯 成员访问：`.first` 和 `.second`

|成员|作用|类型|
|---|---|---|
|`p.first`|获取第一个元素（key）|类型1|
|`p.second`|获取第二个元素（value）|类型2|

```cpp
auto name = student.first;   // "Alice"
auto age  = student.second;  // 20
```

💡 ​**建议**​：用 `auto&` 避免不必要的拷贝：

```cpp
const auto& name = student.first;  // 更高效
```

---

🛠️ 创建方式

|方法|语法|说明|
|---|---|---|
|直接初始化|`pair p{"A", 5};`|C++11 起支持|
|`std::make_pair`|`auto p = std::make_pair("B", 10);`|自动推导类型|
|构造函数|`pair p = {"C", 15};`|支持列表初始化|

---

🔄 比较操作（支持 `<`, `==`, `!=`, `<=`, 等）

按 **字典序** 比较：先比 `first`，再比 `second`。

```cpp
std::pair<int, int> a{1, 2};
std::pair<int, int> b{1, 3};

if (a < b) {  // true → 因为 second: 2 < 3
    std::cout << "a < b\n";
}
```

---

🧩 常见用途场景

|场景|示例|
|---|---|
|函数返回两个值|`std::pair find_key(...)`|
|映射中存储键值对|`std::map` 内部就是 `pair`|
|遍历容器|`for (auto& p : vec_of_pairs)`|
|临时数据打包|`return {name, score};`|

---

⚠️ 注意事项 & 最佳实践

- ✅ 使用 `auto` 或 `auto&` 提高可读性和性能。
- ✅ 避免在频繁调用中创建临时 `pair`，考虑移动语义。
- ❌ 不要用 `pair` 存储大量数据（如大对象），易造成性能开销。
- ✅ 若只关心一个成员，优先使用 `.first` / `.second`，别用 `get<0>(p)`（除非需要模板泛型）。

---

📦 高级技巧（进阶）

```cppcpp
// 使用 std::get<0>(p) —— 适用于元组风格访问
#include <tuple>
std::get<0>(student);  // "Alice"

// 与结构化绑定（C++17）结合使用（最推荐！）
auto [name, age] = student;
std::cout << name << ", " << age << "\n";  // Alice, 20
```

> ✅ 结构化绑定是现代 C++ 的首选写法！

---

📎 总结一句话

> `pair` 就像一个“双胞胎盒子”：  
> ​**`.first` 是左孩，`.second` 是右娃**​，用起来简单又高效！

## 5、`vector` vs `list`（v1.0）

> ✅ 适用于 C++11 及以上标准  
> 📌 所有容器均位于 `<vector>` 和 `<list>`

---

### 🔹 5.1 基本对比总览

|特性|`std::vector`|`std::list`|
|---|---|---|
|底层实现|动态数组（连续内存）|双向链表（非连续内存）|
|插入/删除效率（头部）|❌ O(n)|✅ O(1)|
|插入/删除效率（中间/尾部）|❌ O(n)（可能触发扩容）|✅ O(1)（已知位置）|
|随机访问（下标访问）|✅ O(1)|❌ O(n)|
|内存局部性|✅ 较好（缓存友好）|❌ 差（节点分散）|
|支持 `operator[]`|✅|❌|
|支持迭代器随机移动|✅|❌（只能 ++/--）|
|内存开销|小（仅存储数据）|大（每个节点有前后指针）|

🟢 ​**一句话总结**​：

- 用 `vector`：​**频繁读取、顺序遍历、尾部插入**​。
- 用 `list`：​**频繁在任意位置插入/删除、需保持迭代器有效性**​。

---

### 🔹 5.2 常用成员函数对照表

|操作|`vector`|`list`|说明|
|---|---|---|---|
|构造|`vector v;`|`list l;`|默认构造|
|添加元素|`v.push_back(x);`|`l.push_back(x);`|尾部添加|
|头部插入|`v.insert(v.begin(), x);`|`l.push_front(x);`|`list` 更高效|
|中间插入|`v.insert(it, x);`|`l.insert(it, x);`|`list` 优势明显|
|删除元素|`v.erase(it);`|`l.erase(it);`|`list` 不会移动其他元素|
|删除头/尾|`v.pop_back();` / `v.pop_front();`（⚠️ 无 `pop_front`）|`l.pop_front();` / `l.pop_back();`|`vector` 无 `pop_front`|
|清空|`v.clear();`|`l.clear();`|两者皆可|
|大小|`v.size();`|`l.size();`|两者都有|
|是否为空|`v.empty();`|`l.empty();`|两者都有|
|访问元素|`v[i];` 或 `v.at(i)`|❌ 不能用下标！|`list` 必须通过迭代器|
|迭代器遍历|`for(auto it = v.begin(); it != v.end(); ++it)`|同上|语法一致|

---

### 🔹 5.3 性能对比图（时间复杂度）

|操作|`vector`|`list`|
|---|---|---|
|`push_back`|O(1) 平摊|O(1)|
|`push_front`|O(n)|O(1)|
|`insert(pos)`|O(n)|O(1)（已知迭代器）|
|`erase(pos)`|O(n)|O(1)（已知迭代器）|
|`at(i)` / `[]`|O(1)|❌ 不支持|
|`size()`|O(1)|O(1)（C++11+）|
|内存分配|分配一次大块内存|每次分配小块内存|

> 💡 提示：`list` 的 `size()` 在 C++11 前是 O(n)，现在为 O(1)，但仍不如 `vector` 高效。

🔹 四、典型使用场景推荐

✅ 什么时候用 `vector`？

- 存储固定数量的数据，比如成绩列表、图像像素；
- 需要快速随机访问（如 `v[100]`）；
- 数据主要在末尾增删；
- 对内存使用敏感，希望减少碎片；
- 配合 `std::sort`, `std::find` 等算法使用。

```cpp
std::vector<int> scores = {85, 92, 78, 96};
std::sort(scores.begin(), scores.end()); // OK！
```

---

✅ 什么时候用 `list`？

- 需要在​**任意位置频繁插入/删除**​（尤其是头部）；
- 保留​**迭代器有效性**​（如删除某元素后仍能访问其他元素）；
- 实现队列、栈、缓冲区等数据结构；
- 不关心随机访问，只做顺序遍历。

```cpp
std::list<std::string> tasks;
tasks.push_front("任务A");
tasks.push_back("任务B");
auto it = tasks.begin();
tasks.erase(it); // 迭代器仍有效（除非被删除）
```

---

### 🔹 5.4 注意事项 & 常见误区

|误区|正确做法|
|---|---|
|`list` 可以用 `[]` 访问？|❌ 不行！必须用迭代器遍历|
|`vector` 插入头部很快？|❌ O(n)，应使用 `push_front` 替代（但 `vector` 无此接口）|
|`list` 比 `vector` 更“安全”？|⚠️ 不一定，内存开销大，缓存不友好，速度反而慢|
|`list` 的 `size()` 很慢？|❌ C++11+ 已优化为 O(1)，但仍是 `vector` 的 2~3 倍常数|

---

### 🔹 5.5 进阶技巧 & 混合使用建议

1. **想高效删除中间元素 + 保持迭代器有效？ → 用 `list`**
    
    ```cpp
    auto it = std::find(l.begin(), l.end(), target);
    if (it != l.end()) {
        l.erase(it); // 安全且高效
    }
    ```
    
2. **想快速访问 + 频繁修改？ → 用 `vector`**
    
    ```cpp
    for (int i = 0; i < v.size(); ++i) {
        v[i] *= 2; // 直接访问，快！
    }
    ```
    
3. **不想复制整个容器？→ 使用 `std::move`**
    
    ```cpp
    std::vector<int> v1 = {1,2,3};
    std::vector<int> v2 = std::move(v1); // v1 变为空，v2 拿走资源
    ```
    
4. **需要双向遍历？→ `list` 更方便**
    
    ```cpp
    for (auto it = l.rbegin(); it != l.rend(); ++it) { ... } // 反向遍历
    ```
    

---

🎁 **附加彩蛋：容器选择决策树**

```cpp
是否需要频繁插入/删除头部？
├── 是 → 用 list
└── 否 → 是否需要随机访问？
        ├── 是 → 用 vector
        └── 否 → 用 list（或 deque）
```

> 🔄 ​**补充建议**​：如果既需要随机访问又需要高效插入/删除，考虑 `std::deque`（双端队列），它是 `vector` 和 `list` 的折中方案。

## 6、multimap、unordered_map

### 🔹 6.1 核心对比总览

|特性|`std::map`|`std::multimap`|`std::unordered_map`|
|---|---|---|---|
|是否有序|✅ 是（按 key 排序）|✅ 是（按 key 排序）|❌ 否（哈希无序）|
|键是否可重复|❌ 否（唯一）|✅ 是（允许多个相同 key）|❌ 否（唯一）|
|底层结构|红黑树|红黑树|哈希表（桶）|
|插入/查找/删除平均复杂度|O(log n)|O(log n)|O(1)（平均）|
|最坏情况复杂度|O(log n)|O(log n)|O(n)（哈希冲突）|
|支持 `lower_bound`/`upper_bound`|✅|✅|❌（无序）|
|支持迭代器遍历（有序）|✅|✅|❌（顺序随机）|
|是否可修改 key？|❌（key 不可变）|❌|❌|

> ✅ ​**一句话总结**​：
> 
> - 要求​**有序遍历**​？→ 用 `map` 或 `multimap`
> - 允许​**重复 key**​？→ 用 `multimap`
> - 追求​**极致性能**​（查/插/删）？→ 用 `unordered_map`
> - 不关心顺序，只关心“key 是否存在”？→ 用 `unordered_map`

---

### 🔹 6.2 `std::multimap` 详解

✅ 用途场景

- 存储“同一个键对应多个值”的数据
- 如：按班级统计学生成绩（一个班有多个学生）
- 按时间戳记录事件日志（相同时间点多个事件）  
    ✅ 基本语法

```cpp
#include <map>
std::multimap<std::string, int> grades;

// 插入
grades.insert({"Alice", 90});
grades.insert({"Bob", 85});
grades.insert({"Alice", 95});  // 允许重复 key

// 查找所有与 key 匹配的值
auto range = grades.equal_range("Alice");
for (auto it = range.first; it != range.second; ++it) {
    std::cout << it->second << std::endl;
}
// 输出：90, 95
```

✅ 常用操作

|操作|说明|
|---|---|
|`insert()`|插入键值对（可重复 key）|
|`equal_range(key)`|返回一对迭代器，指向所有相同 key 的范围|
|`count(key)`|返回该 key 出现次数|
|`erase(key)`|删除所有该 key 的元素|
|`find(key)`|找到第一个匹配的 key，返回迭代器（若无则返回 end）|
|`lower_bound(key)`|返回第一个 ≥ key 的位置|
|`upper_bound(key)`|返回第一个 > key 的位置|

> ⚠️ 注意：`find` 只返回第一个匹配项，不返回全部！

---

### 🔹 6.3 `std::unordered_map` 详解

✅ 用途场景

- 快速查找、插入、删除（如缓存、统计词频、ID 映射）
- 适合“键唯一、无需排序”的场景
- 如：用户 ID → 用户信息，单词 → 出现次数  
    ✅ 基本语法

```cpp
#include <unordered_map>
std::unordered_map<std::string, int> word_count;

// 插入或更新
word_count["apple"] = 1;
word_count["banana"]++; // 自增
word_count["apple"] += 5;

// 查找
if (word_count.find("apple") != word_count.end()) {
    std::cout << "apple count: " << word_count["apple"] << std::endl;
}

// 遍历
for (const auto& [key, value] : word_count) {
    std::cout << key << ": " << value << std::endl;
}
```

✅ 常用操作

|操作|说明|
|---|---|
|`insert()`|插入键值对（若 key 已存在则不替换）|
|`emplace()`|就地构造，效率更高|
|`find(key)`|查找 key，返回迭代器|
|`count(key)`|返回 key 的数量（0 或 1）|
|`operator[]`|访问 key，若不存在则插入默认值（如 `int` 为 0）|
|`erase(key)`|删除指定 key 的元素|
|`bucket_count()`|返回哈希桶数量|
|`load_factor()`|返回负载因子（平均每个桶元素数）|

> ✅ ​**小技巧**​：用 `find` 判断是否存在，避免 `operator[]` 无意插入默认值！

```cpp
auto it = word_count.find("orange");
if (it != word_count.end()) {
    std::cout << "Found: " << it->second << std::endl;
} else {
    std::cout << "Not found." << std::endl;
}
```

---

### 🔹 6.4 常见错误 & 避坑指南

|错误|正确做法|
|---|---|
|用 `unordered_map` 期望有序输出？❌|`unordered_map` ​**无序**​！要用 `map` 或 `multimap`|
|用 `map` 做高频查表？⚠️|`unordered_map` 更快！除非需要有序遍历|
|忘记 `const` 引用导致性能下降？⚠️|✅ 所有迭代器访问都用 `const auto&`|
|在 `unordered_map` 中使用自定义类型作为 key？⚠️|✅ 必须提供哈希函数（或特化 `std::hash`）|
|使用 `multimap` 但只用 `find`？⚠️|❌ `find` 只返回第一个，要用 `equal_range` 才能获取全部|
|误以为 `map` 和 `multimap` 可以修改 key？❌|❌ key 一旦插入不可修改！|

> 🔍 ​**特别提醒**​：
> 
> - `unordered_map` 的 key 必须支持 `std::hash` 和 `operator==`
> - 自定义结构体作为 key 时，需重载这两个操作符：

```cpp
struct Point {
    int x, y;
    bool operator==(const Point& other) const {
        return x == other.x && y == other.y;
    }
};

// 哈希函数特化
namespace std {
    template<>
    struct hash<Point> {
        size_t operator()(const Point& p) const {
            return std::hash<int>()(p.x) ^ (std::hash<int>()(p.y) << 1);
        }
    };
}
```

### 🔹 6.5 进阶技巧

✅ 1. `multimap` 与 `map` 的性能对比

|操作|`map`|`multimap`|`unordered_map`|
|---|---|---|---|
|插入|O(log n)|O(log n)|O(1) 平均|
|查找|O(log n)|O(log n)|O(1) 平均|
|删除|O(log n)|O(log n)|O(1) 平均|
|遍历有序|✅|✅|❌|

> ✅ 推荐：
> 
> - 要有序？→ `map` / `multimap`
> - 要快？→ `unordered_map`
> - 要重复键？→ `multimap`  
>     ✅ 2. 如何判断 `unordered_map` 是否存在某个 key？

```cpp
if (my_map.find(key) != my_map.end()) {
    // 存在
}
```

✅ 3. `multimap` 的 `equal_range` 使用模板

```cpp
auto [first, last] = my_multimap.equal_range("Alice");
std::for_each(first, last, [](const auto& p) {
    std::cout << p.second << std::endl;
});
```

## 7、stack、deque、queue

📌 C++ 容器速查卡：`stack`、`queue`、`deque`

> 标准库中的三大常用容器适配器与序列容器，掌握它们能大幅提升算法效率！

---

### 📦 7.1 `std::deque`（双端队列）

💡 什么是 deque？

- 双向开口的动态数组，支持两端高效插入/删除。
- 底层实现：分段连续内存块（类似“链式数组”）。

🛠️ 常用操作

|操作|语法|时间复杂度|说明|
|---|---|---|---|
|头插|`dq.push_front(x)`|O(1)|✅ 高效|
|尾插|`dq.push_back(x)`|O(1)|✅ 高效|
|头删|`dq.pop_front()`|O(1)|✅ 高效|
|尾删|`dq.pop_back()`|O(1)|✅ 高效|
|访问首元素|`dq.front()`|O(1)|返回引用|
|访问尾元素|`dq.back()`|O(1)|返回引用|
|随机访问|`dq[i]`|O(1)|✅ 支持下标|
|大小|`dq.size()`|O(1)|快速获取|
|判空|`dq.empty()`|O(1)|优雅判断|

```
#include <deque>
std::deque<int> dq = {1, 2, 3};

dq.push_front(0);     // [0, 1, 2, 3]
dq.push_back(4);      // [0, 1, 2, 3, 4]
dq.pop_front();       // [1, 2, 3, 4]
dq.pop_back();        // [1, 2, 3]

std::cout << dq.front() << ", " << dq.back(); // 1, 3
```

✅ 适用场景

- 实现滑动窗口（如最大值问题）
- 需要两端频繁插入/删除
- 需要随机访问（比 `list` 快）

---

### 📥 7.2 `std::stack`（栈）

💡 什么是 stack？

- 后进先出（LIFO）的容器适配器，​**默认基于 `deque` 构建**​。

🛠️ 常用操作

|操作|语法|时间复杂度|说明|
|---|---|---|---|
|入栈|`s.push(x)`|O(1)|压入顶部|
|出栈|`s.pop()`|O(1)|移除顶部|
|查看顶元素|`s.top()`|O(1)|返回引用|
|大小|`s.size()`|O(1)|快速获取|
|判空|`s.empty()`|O(1)|优雅判断|

```
#include <stack>
std::stack<int> s;

s.push(1);
s.push(2);
s.push(3);

std::cout << s.top(); // 3
s.pop();              // 移除 3
std::cout << s.top(); // 2
```

⚙️ 自定义底层容器（可选）

```
std::stack<int, std::vector<int>> s_vec;   // 用 vector 实现
std::stack<int, std::list<int>> s_list;    // 用 list 实现
```

✅ 适用场景

- 表达式求值（括号匹配、后缀表达式）
- 函数调用栈模拟
- 回溯算法（DFS）

---

### 📤 7.3 `std::queue`（队列）

💡 什么是 queue？

- 先进先出（FIFO）的容器适配器，​**默认基于 `deque` 构建**​。

🛠️ 常用操作

|操作|语法|时间复杂度|说明|
|---|---|---|---|
|入队|`q.push(x)`|O(1)|加入尾部|
|出队|`q.pop()`|O(1)|移除头部|
|查看队首|`q.front()`|O(1)|返回引用|
|查看队尾|`q.back()`|O(1)|返回引用|
|大小|`q.size()`|O(1)|快速获取|
|判空|`q.empty()`|O(1)|优雅判断|

```
#include <queue>
std::queue<int> q;

q.push(1);
q.push(2);
q.push(3);

std::cout << q.front(); // 1
q.pop();                // 移除 1
std::cout << q.front(); // 2
```

⚙️ 自定义底层容器（可选）

```
std::queue<int, std::list<int>> q_list;  // 用 list 实现
```

✅ 适用场景

- 广度优先搜索（BFS）
- 消息队列、任务调度
- 缓冲区处理

---

### 📊 7.4 对比总结表

|特性|`deque`|`stack`|`queue`|
|---|---|---|---|
|是否支持随机访问|✅ 是|❌ 否|❌ 否|
|支持两端操作|✅ 是|❌ 仅一端|❌ 仅两端|
|插入/删除性能|O(1)（两端）|O(1)（顶端）|O(1)（两端）|
|默认底层|`deque`|`deque`|`deque`|
|适用典型场景|滑动窗口、双向操作|括号匹配、DFS|BFS、任务调度|

---

🎯 最佳实践建议

- ✅ 用 `deque` 当你需要 ​**两端操作 + 随机访问**​。
- ✅ 用 `stack` 实现 LIFO 逻辑，别自己手动维护。
- ✅ 用 `queue` 实现 FIFO 逻辑，别用 `vector` 手动模拟。
- ✅ 优先使用结构化绑定 + `auto` 提升可读性：
    
    ```
    auto [val, idx] = my_pair;
    ```
    
- ❌ 不要在频繁遍历中使用 `stack` 或 `queue`，它们不支持迭代器。

---

📎 总结一句话

> - `deque`：​**全能选手**​，两端自由出入，还能下标访问。
> - `stack`：​**后进先出**​，像一摞书，只能拿最上面那本。
> - `queue`：​**先进先出**​，像排队买票，谁先来谁先走。

## 8、set、multiset、unordered_set

📌 C++ 容器速查卡：`set`、`multiset`、`unordered_set`

> 标准库中的三大集合容器，掌握它们能让你高效去重、排序、查找数据！

---

### 📦 8.1 `std::set`（有序唯一集合）

💡 什么是 set？

- **自动排序**的​**唯一键值集合**​，不允许重复元素。
- 底层实现：​**红黑树**​（平衡二叉搜索树）。
- 所有操作保持有序性。

🛠️ 常用操作

|操作|语法|时间复杂度|说明|
|---|---|---|---|
|插入元素|`s.insert(x)`|O(log n)|若已存在则不插入|
|删除元素|`s.erase(x)`|O(log n)|成功返回 1，否则 0|
|查找元素|`s.find(x)`|O(log n)|返回迭代器，未找到返回 `end()`|
|判断是否存在|`s.count(x)`|O(log n)|返回 0 或 1|
|大小|`s.size()`|O(1)|快速获取|
|判空|`s.empty()`|O(1)|优雅判断|
|遍历|`for (const auto& x : s)`|O(n)|自动按序输出|

```
#include <set>
std::set<int> s = {5, 2, 8, 1};

s.insert(3);         // {1, 2, 3, 5, 8}
s.erase(2);          // {1, 3, 5, 8}

if (s.find(5) != s.end()) {
    std::cout << "Found 5\n";
}

for (int x : s) {
    std::cout << x << " "; // 输出：1 3 5 8
}
```

✅ 适用场景

- 需要**自动排序 + 去重**
- 动态维护有序数据集（如排行榜）
- 查找是否存在某个元素（比 `vector` 快得多）

---

### 📦 8.2 `std::multiset`（有序多重集合）

💡 什么是 multiset？

- 类似 `set`，但允许​**重复元素​**​。
- 仍保持​**排序顺序**​，底层也是红黑树。  
    🛠️ 常用操作（与 `set` 基本一致）

|操作|语法|时间复杂度|说明|
|---|---|---|---|
|插入元素|`ms.insert(x)`|O(log n)|可重复插入|
|删除所有匹配元素|`ms.erase(x)`|O(log n + k)|删除所有值为 `x` 的元素|
|删除指定位置元素|`ms.erase(it)`|O(log n)|单个删除|
|查找第一个匹配元素|`ms.find(x)`|O(log n)|返回第一个|
|统计出现次数|`ms.count(x)`|O(log n + k)|返回数量|
|大小|`ms.size()`|O(1)|快速获取|
|遍历|`for (const auto& x : ms)`|O(n)|按序输出，含重复项|

```
#include <set>
std::multiset<int> ms = {1, 2, 2, 3, 3, 3};

ms.insert(2);        // {1, 2, 2, 2, 3, 3, 3}
ms.erase(2);         // 删除所有 2 → {1, 3, 3, 3}

std::cout << ms.count(3); // 输出：3

for (int x : ms) {
    std::cout << x << " "; // 输出：1 3 3 3
}
```

✅ 适用场景

- 统计频次（如词频统计）
- 需要保留重复元素且仍需排序
- 排名系统中允许并列（如考试成绩）

---

### 📦 8.3 `std::unordered_set`（无序唯一集合）

💡 什么是 unordered_set？

- **哈希表实现**的集合，​**不保证顺序**​，但​**平均查找/插入/删除为 O(1)**​。
- 不允许重复元素。
- 依赖于哈希函数，需支持 `std::hash<T>` 或自定义。  
    🛠️ 常用操作

|操作|语法|时间复杂度|说明|
|---|---|---|---|
|插入元素|`us.insert(x)`|**平均 O(1)**|最坏 O(n)，但通常很快|
|删除元素|`us.erase(x)`|**平均 O(1)**|成功返回 1，否则 0|
|查找元素|`us.find(x)`|**平均 O(1)**|返回迭代器，未找到返回 `end()`|
|判断是否存在|`us.count(x)`|**平均 O(1)**|返回 0 或 1|
|大小|`us.size()`|O(1)|快速获取|
|判空|`us.empty()`|O(1)|优雅判断|

```
#include <unordered_set>
std::unordered_set<std::string> us = {"apple", "banana", "cherry"};

us.insert("date");           // 添加
us.erase("banana");          // 删除

if (us.find("apple") != us.end()) {
    std::cout << "Apple exists!\n";
}

std::cout << us.count("grape"); // 输出：0
```

⚠️ 注意事项

- ​**不能保证顺序**​，遍历时顺序不确定。
- 适用于**大量快速查找、去重**场景。
- 若哈希冲突严重，性能可能退化为 O(n)。
- 自定义类型需提供 `std::hash<T>` 特化或传入哈希函数。

```
struct Point {
    int x, y;
    bool operator==(const Point& p) const {
        return x == p.x && y == p.y;
    }
};

struct HashPoint {
    size_t operator()(const Point& p) const {
        return std::hash<int>()(p.x) ^ (std::hash<int>()(p.y) << 1);
    }
};

std::unordered_set<Point, HashPoint> points;
```

✅ 适用场景

- 快速判断元素是否已存在（如去重、判重）
- 大规模数据去重（如日志分析、URL 去重）
- 无需排序的唯一集合

---

### 📊 8.4 对比总结表

|特性|`set`|`multiset`|`unordered_set`|
|---|---|---|---|
|是否有序|✅ 是（升序）|✅ 是（升序）|❌ 否（无序）|
|是否允许重复|❌ 否|✅ 是|❌ 否|
|查找/插入/删除|O(log n)|O(log n)|**平均 O(1)**|
|空间开销|较高（树节点）|较高|中等（哈希桶）|
|内存稳定性|高（不迁移）|高|低（可能重新哈希）|
|适用典型场景|排序+去重|统计频次+排序|快速查重、海量数据|

---

🎯 最佳实践建议

- ✅ 用 `set` 当你需要 ​**自动排序 + 去重**​。
- ✅ 用 `multiset` 当你需要 ​**保留重复 + 排序**​（如频次统计）。
- ✅ 用 `unordered_set` 当你需要 ​**极致查找速度 + 不关心顺序**​。
- ✅ 优先使用 `count()` 或 `find()` 判断存在性，不要用 `find()` + `end()` 比较。
- ✅ 避免在 `unordered_set` 中存储指针或复杂对象（除非提供良好哈希）。
- ✅ 使用 `emplace()` 而非 `insert()`，减少拷贝开销：
    
    ```
    us.emplace("hello");
    ```
    

---

📎 总结一句话

> - `set`：​**有序去重**​，像一本排好序的字典。
> - `multiset`：​**有序允重**​，像一个带重复条目的名单。
> - `unordered_set`：​**无序去重**​，像一个快速查重的哈希表。

## 9、tuple

📋 C++ `std::tuple` 速查表（v1.0）

> ✅ 适用于 C++11 及以上标准  
> 📌 重点：**元组（tuple）—— 多个不同类型值的聚合容器**

---

### 🔹9.1 核心特性总览

|特性|说明|
|---|---|
|**类型安全**|编译期类型检查，防止类型错误|
|**异构存储**|可存放不同类型（int, string, double, 自定义类等）|
|**固定大小**|创建后大小不可变（编译期确定）|
|**支持索引访问**|通过 `std::get(tuple)` 访问|
|**支持解包（unpacking）**|与结构化绑定（C++17）完美结合|
|**不支持动态扩展**|不能增删元素|

> ✅ ​**一句话总结**​：  
> `tuple` 是“​**类型各异、数量固定、可命名/可索引**​”的复合数据结构，适合返回多个值、封装配置、函数参数打包等。

---

### 🔹9.2 基本语法与创建

✅ 1. 声明与初始化

```
#include <tuple>
#include <string>

// 方式一：直接构造
std::tuple<int, std::string, double> t1 = {42, "Hello", 3.14};

// 方式二：使用 make_tuple
auto t2 = std::make_tuple(100, "World", 2.718);

// 方式三：使用 make_tuple + 类型推导（C++14 起）
auto t3 = std::make_tuple("C++", 2023, true);
```

> ✅ 推荐使用 `make_tuple`，可自动推导类型，避免冗长。

---

✅ 2. 获取元素：`std::get<index>(tuple)`

```
auto t = std::make_tuple(10, "Hello", 3.14);

std::cout << std::get<0>(t) << std::endl;  // 输出：10
std::cout << std::get<1>(t) << std::endl;  // 输出：Hello
std::cout << std::get<2>(t) << std::endl;  // 输出：3.14
```

> ⚠️ 注意：索引从 `0` 开始，​**必须在编译期确定**​，不能用变量！

---

### 🔹9.3 结构化绑定（C++17）—— 最佳实践！

✅ 1. 语法：`auto [a, b, c] = tuple;`

```
auto t = std::make_tuple(100, "C++", 2.0);

// 解包到局部变量
auto [num, msg, pi] = t;

std::cout << num << " " << msg << " " << pi << std::endl;
// 输出：100 C++ 2
```

✅ 2. 结构化绑定的高级用法

```
// 与 std::tie 结合（C++11）
std::tuple<int, std::string> result = {42, "success"};
int code;
std::string msg;
std::tie(code, msg) = result;

// 与 auto 结合（C++17）
auto [code2, msg2] = result;
```

> ✅ ​**强烈推荐**​：在 C++17 及以上环境中，优先使用结构化绑定，简洁清晰！

---

### 🔹9.4 常用操作与函数

|操作|说明|
|---|---|
|`std::make_tuple()`|构造 tuple，支持类型推导|
|`std::tie()`|解包 tuple 到变量（可用于赋值）|
|`std::get(tuple)`|按索引访问元素|
|`std::tuple_size`|获取 tuple 元素个数（编译期）|
|`std::tuple_element`|获取第 index 个元素的类型|
|`std::make_from_tuple()`|从 tuple 构造对象（C++17）|
|`std::apply()`|将 tuple 作为参数调用函数（C++17）|

✅ 示例：`std::apply` 用法

```
#include <functional>

void print(int a, const std::string& b, double c) {
    std::cout << a << ", " << b << ", " << c << std::endl;
}

auto t = std::make_tuple(10, "Hello", 3.14);
std::apply(print, t);  // 输出：10, Hello, 3.14
```

> 🎯 用途：将 tuple 作为参数传递给函数，避免手动展开。

---

### 🔹9.5 高级技巧与实用场景

✅ 1. 返回多个值（替代 `std::pair`）

```
std::tuple<int, std::string, bool> process_data() {
    return {42, "Success", true};
}

auto [code, msg, ok] = process_data();
```

> ✅ 比 `std::pair` 更灵活，支持任意数量的返回值。

---

✅ 2. 函数参数打包（与 `std::apply` 配合）

```
void log(int level, const std::string& msg, double time) {
    std::cout << "Level " << level << ": " << msg << " (" << time << "s)" << std::endl;
}

auto args = std::make_tuple(2, "Error", 0.5);
std::apply(log, args);
```

> ✅ 适合日志、事件分发、回调封装等场景。

---

✅ 3. 模板元编程辅助（类型萃取）

```
template<typename T>
void print_tuple_size() {
    std::cout << "Size: " << std::tuple_size<T>::value << std::endl;
}

// 使用
print_tuple_size<std::tuple<int, double, std::string>>(); // 输出：3
```

> ✅ 用于编译期类型检查、模板编程。

---

### 🔹9.6 常见错误 & 避坑指南

|错误|正确做法|
|---|---|
|用变量索引 `std::get(t)`|❌ `i` 必须是编译期常量！|
|忘记 `#include`|✅ 确保包含头文件|
|误以为 tuple 支持动态扩容|❌ tuple 大小固定，不能增删元素|
|用 `std::get<0>(t)` 但类型不匹配|❌ 编译期检查，类型必须一致|
|无法解包非 const tuple|✅ 使用 `const auto&` 或 `auto&&`|

---

### 🔹9.7 速查表使用建议

- 打印成 A4 纸，贴在桌边
- 与 `std::pair`、`std::array`、`std::vector` 一起构成“标准容器家族”
- 面试高频题：如何返回多个值？如何解包？如何调用函数？

---

🎁 **附加彩蛋：面试高频题 + 解答**

1. **Q：`std::tuple` 和 `std::pair` 的区别？**  
    A：`pair` 只能存两个元素，`tuple` 可存任意数量；`tuple` 更灵活。
2. **Q：如何在 C++17 中解包 tuple？**  
    A：使用结构化绑定：`auto [a, b, c] = t;`
3. **Q：`std::apply` 是做什么的？**  
    A：将 tuple 作为参数调用函数，避免手动展开。
4. **Q：`std::get<index>(t)` 为什么不能用变量？**  
    A：索引必须是编译期常量，因为 `get` 是模板函数。

📌 ​**一句话总结**​：

> `tuple` 是“​**多类型、固定大小、可解包**​”的复合数据结构，  
> 用 `make_tuple` 创建，用 `get` 或结构化绑定访问，  
> 配合 `apply` 可实现函数调用解包，是现代 C++ 的利器！
