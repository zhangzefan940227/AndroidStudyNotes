# 1 前言
在 C++ 中，`*`（星号）和 `&`（艾特符/取地址符）确实是初学者最容易混淆的魔鬼符号。它们之所以让人混乱，是因为==**同一个符号在“声明语句”和“执行语句”中的含义完全相反**==。

要彻底分清它们，秘诀在于**先看这个符号出现在什么位置**。我们可以把场景切分为两大世界：**【声明世界】**（定义变量、函数参数、返回值）和**【执行世界】**（具体的业务代码、表达式）。

🌟 核心分清口诀

- **在声明世界里**：`*` 表示**指针**（Pointer），`&` 表示**引用**（Reference）。它们是在**定义类型**。
- **在执行世界里**：`*` 表示**解引用**（去拿指针指向的值），`&` 表示**取地址**（去拿变量的内存地址）。它们是**动作运算符**。

# 3 详细解释
## 3.1 声明世界
### `*` 符号
**声明变量**
- 含义：表示这是一个**指针变量**，用来存放某种类型的**内存地址**
- 代码示例：
```cpp
int *p; // 读法：p是一个指针(*)，存放的是地址，该地址指向一个int类型的值
```


**函数参数**
- 含义：表示**指针传递**（本质上还是声明一个指针参数），函数接收的是一个地址。在函数内部修改它，会影响外部真实的变量。
- 代码示例：
```cpp
void updateValue(int* p) { 
    *p = 20; // 修改了外部变量的值
}
```

**函数返回值**
- 含义：表示函数返回的是一个**内存地址（指针）**，而不是具体的值。
- 代码示例：
```cpp
int* getPointer(int& num) {
    return &num; // 返回了 num 的地址
}
```
个人理解：
`*`修饰的指针变量中存放的永远是地址。返回通过取地址符`&`获取的内存地址符合函数预期。

### `&` 符号
**声明变量**
- 含义：表示这是一个**引用（别名）**。它不是新变量，它只是某个已有变量的“外号”，它们共享同一块内存。
- 代码示例：
```cpp
int a = 10;
int& ref = a; //声明世界：ref是a的引用，改ref就是改a
```

**函数参数**
- 含义：表示引用传递。**这是 C++ 中最推荐的传参方式。** 既能像指针一样在函数内修改外部变量，又不需要写麻烦的 `*` 和 `&`，还能避免复制大对象的内存损耗。
- 代码示例：
```cpp
void swap(int& x, int& y) { // 声明世界：x 和 y 是传入变量的别名
    int temp = x;
    x = y;
    y = temp; // 直接交换了外部真实变量的值
}
```
- 个人理解：
    引用作为参数的时候，传入数据时，可以看做是给引用形参初始化。
    比如上述例子中，调用接口应传入具体参数，这里就可以看做给实参起了个别名：
```cpp
int a = 10;
int b = 20;
swap(a, b);  // 可以看做 int& x = a; int& y = b; 修改x y的值实际就是修改a b的值
```

**函数返回值**
- 含义：表示函数返回的是一个**变量的引用（别名）**。由于返回的是变量本身，所以这个函数调用甚至可以放在赋值号 = 的左边（称为左值）。
- 代码示例：
```cpp
int global_num = 10;
int&getGlobal() { 
    return global_num; // 返回全局变量的引用
}

// 在 main 中：
getlglobal() = 50; // 极其神奇：直接把 global_num 改成了 50
```

## 3.2 执行世界
### `*` 符号
- 含义：**解引用**。意思是：”去这个地址里，把里面的值拿出来。”
- 代码示例：
```cpp
int a = 10;
int* p = &a; // 此时 p 存着 a 的地址
int b = *p;  // 执行世界：*p 意思是去 p 的地址拿值，所以 b 变成了 10
*p = 100;    // 执行世界：把 100 写入 p 指向的内存，此时 a 变成了 100
```

### `&` 符号
- 含义：**取地址**。意思是：“把这个变量在内存里的十六进制**地址**给我拿出来”。
- 代码示例：
```cpp
int a = 10;
std::cout << &a; // 执行世界：打印出 a 的内存地址（如 0x7ffee3bf57bc）
int* p = &a;     // 执行世界：取 a 的地址赋给指针 p
```


# 4 智能指针
智能指针是为了防止内存泄漏问题。

## 4.1 unique_ptr

`unique_ptr` 实现了独占式拥有概念，保证同一时间内只有一个智能指针可以指向该对象，有效避免了资源泄露。unique_ptr不允许拷贝和赋值，这是为了确保资源的独占性 。

代码示例：
```cpp
#include <iostream>
#include <memory>

int main() {
    std::unique_ptr<int> u1(new int(10));
    // std::unique_ptr<int> u2 = u1;  // 编译错误，不允许拷贝
    std::unique_ptr<int> u3 = std::move(u1); 
    if (!u1) {
        std::cout << "u1 is nullptr" << std::endl;
    }
    std::cout << *u3 << std::endl; 
    return 0;
}
```

在这段代码中，尝试将u1赋值给u2会导致编译错误，而通过std::move将u1的所有权转移给u3后，u1变为空指针，u3拥有了资源的所有权 。

unique_ptr支持管理数组，它会使用delete[]来释放数组内存 。同时，unique_ptr还可以作为函数返回值，高效地将资源的所有权转移给调用者。

代码示例：
```cpp
#include <iostream>
#include <memory>

std::unique_ptr<int> createInt() {
    return std::unique_ptr<int>(new int(20));
}

int main() {
    std::unique_ptr<int> ptr = createInt();
    std::cout << *ptr << std::endl; 
    return 0;
}
```

在上述代码中，createInt函数返回一个`unique_ptr<int>`，将资源的所有权转移给main函数中的ptr 。
